Homework 1: Tasks for fl130

Your personal task is to develop a system for markets and their stalls.

Task 1: Configuration [4 marks]

Configure the project to support the use of JSPs as view technology. JSPs must be configured to be placed inside folder src/main/webapp/WEB-INF/views/.

Task 2: Domain Classes [13 marks]

Create class co2123.hw1.domain.Stall. With the following private fields:

  1. String type
  2. String owner
  3. String products
  4. int hours
Generate necessary getters and setters. [7 marks]

Create class co2123.hw1.domain.Market. With the following private fields:

  1. int id
  2. String name
  3. String location
  4. List<Stall> stalls
Generate necessary getters and setters. [6 marks]

Task 3: Controller (simple) [5 marks]

Create controller co2123.hw1.controller.MainController [2 marks].

  1. Requests for URL "/" (POST and GET) should show the view start (the corresponding JSP already exists). [3 marks]

Task 4: Controller, View, and Form [20 marks]

Create field public static List<Market> markets in class co2123.hw1.Hw1Application. Initialize the list inside class co2123.hw1.Hw1Application to contain one market with two stalls. [4 marks]

Create controller co2123.hw1.controller.MarketController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/markets" should show view markets/list. [2 marks]
    • Add the list of all markets from co2123.hw1.Hw1Application to the model as attribute markets. [2 marks]
    • Create the corresponding JSP to list the name and location of each market followed by a link to URL "/stalls?market={id of market}", e.g., "/stalls?market=3". [4 marks]
  2. Requests for URL "/newMarket" should show view markets/form. [3 marks] Create the corresponding JSP with a form with modelAttribute="market" and action="/addMarket" and inputs for all fields of class Market except for stalls. [5 marks]
  3. POST requests for URL "/addMarket" should add a new market to the list of all markets in co2123.hw1.Hw1Application and redirect to /markets. [3 marks]

Task 5: Controller, View, and Form with additional Parameter [33 marks]

Create controller co2123.hw1.controller.StallController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/stalls" should show view stalls/list. [2 marks]
    • Based on request parameter market determine the corresponding market from co2123.hw1.Hw1Application. [2 marks]
    • Add the list of all stalls of that market to the model as attribute stalls. [2 marks]
    • Create the corresponding JSP for the view to list the type, owner, products, and hours of each stall. [4 marks]
    • At the end of the list in the JSP add a link to URL "/newStall?market={id of market whose stalls are listed}", e.g., "/newStall?market=3". [4 marks]
  2. Requests for URL "/newStall" should show view stalls/form. [3 marks]
    • Create the corresponding JSP with a form with modelAttribute="stall" and action="/addStall" and inputs for all fields of class Stall. [6 marks]
    • Also add the request parameter market with the ID of the current market. [4 marks]
  3. POST requests for URL "/addStall" should add a new stall to the list of stalls of the market identified by request parameter market and then redirect to /markets. [6 marks]

Task 6: Validation [25 marks]

Set up a validator for class Market and make it available in controller MarketController. [4 marks]

When adding a new market validate that

  1. The value of field id is unique within the list Hw1Application.markets. [3 marks]
  2. The value of field name is not empty. [2 marks]
  3. The value of field location is not empty. [2 marks]
In case of validation errors go back to the form and show error messages for each field in the form.

Set up a validator for class Stall and make it available in controller StallController. [4 marks]

When adding a new stall validate that

  1. The value of field type is not empty. [2 marks]
  2. The value of field owner is not empty. [2 marks]
  3. The value of field products is either "Fruit", "Vegetables", or "Flowers". [3 marks]
  4. The value of field hours is between (including) 12 and 20. [3 marks]
In case of validation errors go back to the form and show error messages for each field in the form.
